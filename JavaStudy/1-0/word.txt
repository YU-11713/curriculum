■ IT
IT:情報を取得、加工、保存、伝送するための科学技術。

ICT；情報技術通信でITと同様。

IoT:様々な物をインターネットに繋ぐこと。

■ OS
OS：パソコンの基本動作（キーボード・マウスなど）

Windows：マイクロソフトが提供しているOS

macOS (Mac OS X)：アップルが提供しているOS

UNIX：AT＆T社ベル研究所で開発されたOS

Linux：最も普及しているオープンソースで改変、導入、導出が可能なOS

■ 言語・環境
マークアップ／フロントエンド：マークアップ言語の特殊表記で見栄えをよくする。/フロントエンド：利用者がソフトウェアを直接触れる部分。

HTML：webサイトを作るために必要なマークアップ言語。

CSS：HTMLを作成した後、デザインする為の要素。

JavaScript：HTMLで作ったWEBサイトの仕組みや動作など。

サーバーサイド：サーバーでプログラムの実行・管理

.NET（C#, VB）：マイクロソフト社が開発したプログラミング言語(主にwindows用）

Java：旧サン・マイクロシステムズ社が開発、米オラクル社が開発を引き継いでいる。ポータブル言語

Java SE：javaの基本機能をまとめたもの

Java EE（Jakarta EE）：JavaSE＋拡張機能

Perl:UNIX系OSを中心に広く普及している言語。

PHP:実行環境webサーバに組み込んで利用されることが多い。

Ruby：処理系としてソースコードをそのまま実行に移せる。
■ サーバーソフト
Apache:安定性や機能が豊富で、企業や個人まで様々な用途で使われる。

nginx：並行処理とメモリ消費の抑制を重視した設計で同時アクセス数が増えても安定的に動作する。

IIS：米マイクロソフトのosであるWindows Serverシリーズに同梱されているWebサーバソフトウェア。

■ SQL
SQL：管理や操作を行うための問い合わせ言語の一つ。

DB2：リレーショナルデーターベース管理システムの一つ。

MySQL：人気の高いオープンソースのRDBMSの一つ。

Maria DB：MySQLのソースを元に開発しているRDBMS。

Oracle DB：オラクルが開発・販売しているRDBMS。

PostgreSQL：MySQLより機能の豊富さや拡張性の高さに定評がある。

SQLite：シンプルなRDBMSの一つ。

SQL Server：マイクロソフト社が開発・販売しているRDBMS。

CRUD：データの作成、読み出し、更新、削除の基本機能。

トランザクション：互いに関連・依存する複数の処理を1つで管理する為に用いる。

コミット：処理や変更などを確定させる、反映させるといった意味。

ロールバック：システム障害が起きた際、過去の正常な状態に戻す。

排他制御（排他ロック）：プログラムの整合性を守るために、他のプログラムの書き込みを禁止するという制御が行われる。

楽観ロック；データそのものに対してロックをかけるのではなく、更新対象のデータが取得時で同じであるか比べる為、整合性を保証する。

悲観ロック：ロックされたレコードは、トランザクションがロールバックされるまで、他のトランザクションから更新されないため、データの整合性を保証する。

実行計画：文を実行するためにOracle Databaseが使用するステップの組合せの事。

チューニング：ソフトウェアの設定や構成を調節して、性能を最大限引き出す作業の事。

パフォーマンス：コンピュータなどの機器やソフトウェア、システムなどの処理性能や実行速度、通信回線・ネットワークなどの伝送速度・容量などのことを指すことが多い。

スロークエリ：データベースにおいて時間のかかっているSQLの事。

クエリログ：データベース動作記憶

IDE（統合開発環境）：コンパイラ、コードの不正を発見・修正するデバッガなどのプログラム開発ツールをひとまとめにした開発環境のことをいう。

Eclipse：JAVAのオープンソースIDE

NetBeans：JAVAのオープンソースIDE

SpringToolSuite：Springベースのアプリケーション開発を行うために提供される開発ツールです。使い勝手はEclipseと同様です。

DBツール：データベース管理システム (DBMS) や関係データベース管理システム (RDBMS) 上のデータを扱うためのツールです。

A5:SQL Mk-2：複雑化するデータベース開発を支援するために開発されたフリーのSQL開発ツールです。

pgAdmin：PostgreSQLに慣れる為に、pgAdminによるGUIを使って覚えていくと良い。

DBeaver：全文検索エンジンの Elasticsearch や Hadoopの上構築されたデータウェアハウス である Apache Hive など様々なデータ分析ツールにも対応している。

MySQL Workbench：MySQLをGUIでわかりやすく操作するツール

Sequel Pro：MacはHiediSQL対応していないためSequel Proを使う。

TeamSQL：終了したツール

■ 肩書
PM：プロジェクトマネージャプロジェクトの作成・遂行・に責任を負う管理者

PL：プロジェクトリーダー（PMのボス）

PG（PE）：ソフトウェア開発ツールなどを用いてコンピュータプログラムを作成・修正する技術者のこと。
SE：ソフトウェアの開発に携わる技術者の内、プログラミング以外の業務を担当する者の事。

デザイナー：視覚的な設計を行う

エンジニア：工学の知識や技術を利用して、便利で快適な環境を作る。

フロントエンド・エンジニア：利用者への表示や操作の受け付けを行う部分を担当する職種

サーバーサイド・エンジニア：サーバー側で動作するプログラムの開発やデータ処理などをおこなうエンジニアの事

バックエンド・エンジニア：サーバーエンジニアにデータベースが加わったエンジニア

■ UI（ユーザーインターフェース）
UI：機器やソフトウェア、システムなどとその利用者の間で情報をやり取りする仕組み。

CUI：文字による表示や入力を基本とする

GUI：絵や図形による表示、画面内の位置を指示することによる操作を基本とする
■ ファイル／ディレクトリ
ファイル：保存、読み込み、移動、削除などする際に一つのまとまりとして取り扱うデータの集合

拡張子：ファイル名「.」の右側部分

ディレクトリ：多数の対象をその所在などの情報と共に一覧できるよう整理したもの

ルート・ディレクトリ：

カレント・ディレクトリ

パス：コンピュータ内で特定の資源の所在を表す文字列のことをパスという。

絶対パス：階層構造の頂点（最上位階層）からの位置関係を記述する方式。

相対パス：現在位置からの相対的な位置関係を記述する方式。

■ ネットワーク
LAN：限られた範囲でデータ通信が使える。

WAN：広範囲ネットワーク

クライアント：サーバーが提供している情報を受信している機器。

サーバー：機能やサービスの提供

■ 各種プロトコル
HTTP：WebサーバとWebクライアントの間でデータの送受信を行うために用いられるプロトコル（通信規約）

FTP：ファイル転送を行うことができるプロトコル（通信規約）の一つ。

HTTPS：Webのデータ転送に用いられるHTTPが、SSLやTLSで暗号化されている状態を表したもの。

SSL：IPネットワークでデータを暗号化して送受信するプロトコル（通信手順）の一つ。

SSH：ネットワークを介して別のコンピュータにログインして操作するためのソフトウェアの一つ。

ポート、ポート番号：インターネットで標準的に用いられるプロトコル（通信規約）であるTCP/IPにおいて、同じコンピュータ内で動作する複数のソフトウェアのどれが通信するかを指定するための番号。

HTTP、FTP、SSHの各ポート番号：TCP/80はHTTP、TCP/20～21はFTP、TCP/22はssh

ファイアウォール：内外の通信を中継・監視し、外部の攻撃から内部を保護するためのソフトウェアや機器、システムなどのこと。

TCP：IPの一段階上位層のプロトコル（通信規約）として標準的に使われるものの一つ。

■ IPアドレス
IPアドレス：通信機器一台ごとに割り当てられた識別番号

グローバルIPアドレス：インターネットに直に割り振られた識別番号

プライベートIPアドレス：企業など組織の内部で運用されるネットワーク上で各機器に割り当てられるIP
アドレス。

■ ドメイン
ドメイン：インターネット上に存在するコンピュータやネットワークを識別し、階層的に管理するために登録された名前のこと

DNS：日常使っている言語の文字を組み合わせた認識しやすいドメイン名でアクセスすること。

VPN：企業内ネットワークも拠点間接続などに使われている。

ブリッジ：複数の宛先情報を参照して中継の可否を判断する機能

プロキシサーバー：インターネットに接続できないコンピュータの代わりに接続を行うサーバ

■ 電子メール
SMTP：メールを相手のメールサーバーまで届ける仕組み

POP：メールソフトが自動的にメールサーバーにアクセスしてメールをダウンロード

IMAP：POPと同じでその違いは、メールをサーバーに残すか残さないか

■ セッションとクッキー
セッション：通信の開始から終了

クッキー：アカウント情報や設定を一時的にブラウザーに保存する。

キャッシュ：ブラウザーによく使われる静的なデータを一時的に保存する。

■ DB（データベース）について

インデックス：収集・格納された大量のデータをすばやく検索・抽出するために作成された索引データ

■ テストについて

フロー：命令や分岐など実行されるかを確認するテスト

■ チケット管理ツール

タスク：同時並行に実行できるOSの機能や構造を「マルチタスク」
　　　　同時に一つのプロセスしか実行できないことを「シングルタスク」

■ バージョン管理

Git：オープンソースの分散バージョン管理システムの一つ。

SVN：リポジトリと呼ばれるサーバ上の領域でファイルを集中的に管理し、複数の作業者が並行して内容を更新していくことができるようにする。

デグレ：新しいバージョンのソフトウェアの品質が、以前より悪くなること。

マージツール：MDLファイル中の既存のマップ式を保持しながら，FDLファイル又はDTDファイルの変更内容をMDLファイルにマージする機能。
